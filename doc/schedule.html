TALK SCHEDULE WITH ACT

Once you have enough accepted talks, you can start working out the talk schedule.

Here are a few general tips:

  • Print the talk list, try to find a few general groups, and try to group them in tracks. Ideally, someone interested in say, XML, or Security should be able to follow all talks related to their pet subject without changing rooms.

  • You should know your speakers or their subject enough to decide which speakers/talks get the big room.

  • Leave 5 minutes between talks. It always takes the speakers some time to set up their material. It also takes time to kick the previous speaker out. ;-)

  • Count 1 minute between two lightning talks (you need time to swap speakers) So, that's ten 5 minutes lightning talks per hour.

  • Do not pit any talker against the lightning talks. The lightning talks will win.

Once you've worked out a rough draft of the schedule, you can start using Act to edit the talks date, time and room. You can also change the duration, but discuss it with the speaker first.

THE SCHEDULE

How it looks

People with the orga bit set can always see (and edit) the schedule. It becomes public when the [talks] item named show_schedule is set to 1.

The first time you'll look at the schedule, it'll be ugly: all the talks are schedule to a non existing room at the same time. This will change once you assigned a room to each talk.

The HTML table uses as many styles as you have rooms, plus three. Imagine you have three rooms, the table will use the following styles: header, out, venue, r1, r2 and r3. Please note that you do not need to define your own CSS for this: simply load the style located at /css/schedule.css with <link rel="stylesheet" type="text/css" href="/css/schedule.css" />.

We'll add new colours when a conference has more than three rooms.

How it works

Act can lay out the schedule when there are talks with a date and room defined. It should gracefully show overlapping talks, so that you can detect there is a problem.

Events are basically talks without a speaker. Only events can happen in rooms out and venue.

Rooms out and venue are special in the sense that events occuring there prevent talks from happening and actually "cut" any overlapping talk in two. One part before the event (usually a coffee break) and the remaining time after.

Only hour-long tutorial should be cut by events.

There are some cases for which Act does not display the schedule properly (this should not happen once your schedule is finished, since you shouldn't have any overlapping talks). Please let us know when this happens.

Events are:

  • the coffee breaks, lunch

  • anything happening outside the conference (pub crawls, etc.)

  • special talks without a defined speaker.

Lightning talks sessions

Lightning talks sessions are fully handled by Act. One can submit a lightning talk, which will be accepted or not, as for the other talks. Lightning talks do not have an individual date, time and duration: you'll have to create one or several "lightning talks" events (or talks if you want the name of the master of ceremony to appear in the schedule) in which you'll paste the list of lightning talks.

It's rather easy to link talks from within an event or talk. Simply note the talk id and use it to create a link to the user and talk in the event or talk description.

As an exemple, here is the source description of the lightning talks session of the French Perl Workshop 2005:

    Programme des présentations éclairs :
    - talk:101
    - talk:63
    - talk:201
    - talk:198
    - talk:190
    - Alexandre Buisse - Le GC de Parrot
    - Pierre Weis - Présentation d'OCaml
    - Stéphane Payrard - GreaseMonkey

(The last three talks were last minutes talks that were not registered in the database.)
